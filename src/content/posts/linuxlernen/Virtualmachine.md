## 1. 安装与初始登录

当 Red Hat Enterprise Linux 成功安装后，系统会提示可以开始使用，并进入系统登录界面 。

:::tip
**建议新手使用 root 用户登录**
普通用户（例如 `lee`）虽然在日常使用中更安全，但初学阶段不建议使用普通用户 。因为普通用户的权限太小，在这个阶段如果遇到未知的问题往往无法自行解决 。
:::

* **超级用户：** 所有的 Linux 的管理员都叫 `root` 。
* **登录方式：** 在登录界面选择“未列出?”，然后输入用户名 `root` 及对应的密码即可进入系统 。

## 2. 注销当前用户

如果需要离开或者切换用户，可以通过系统自带的图形化菜单注销：

1.  点击屏幕右上角的系统状态区域（一般包含网络、音量、电池图标等） 。
2.  在弹出的菜单中，展开“关机/注销”选项 。
3.  点击“注销”，或者你也可以选择“切换用户” 。

## 3. 虚拟机的克隆 (VMware Workstation)

在进行危险操作或复杂实验前，先克隆一份虚拟机是非常好的习惯。克隆向导将帮助您创建此虚拟机的副本 。

:::note
**开启模板模式：** 如果您想让其他用户创建此虚拟机的克隆，应在“虚拟机” > “设置” > “选项” > “高级”下启用模板模式 。
:::

### 克隆步骤：
1.  在 VMware Workstation 左侧列表中，右键点击需要克隆的虚拟机，选择“管理” -> “克隆” 。
2.  进入**欢迎使用克隆虚拟机向导**，点击“下一页” 。
3.  **选择克隆源：** 一般选择从“虚拟机中的当前状态”创建克隆 。
4.  **选择克隆方法：**
    * **创建链接克隆：** 这是对原始虚拟机的引用，所需的存储磁盘空间较少。但是，必须能够访问原始虚拟机才能运行 。
    * **创建完整克隆：** 原始虚拟机当前状态的完整副本。此副本完全独立，但需要较多的存储磁盘空间 。
5.  **命名与位置：** 为新虚拟机输入名称（例如 `hosta`），并选择一个本地存储位置（例如 `D:\hosta`），最后点击“完成” 。

## 4. 虚拟机的删除

如果某个实验用的虚拟机不再需要了，可以将其彻底删除释放硬盘空间。

1.  在 VMware Workstation 左侧的“我的计算机”列表中，找到要删除的虚拟机（例如 `hosta`） 。
2.  右键点击该虚拟机名称，在弹出的菜单中选择“管理” 。
3.  点击“从磁盘中删除”即可 。

:::caution
**注意区分“移除”与“从磁盘中删除”：**
点击“从磁盘中删除”会将虚拟机的所有文件从你的物理硬盘上彻底清空，数据无法恢复 。如果是点击“移除”，则仅仅是从 VMware 的左侧列表里隐藏它，虚拟机文件依然在你的硬盘上占用空间 。
:::
